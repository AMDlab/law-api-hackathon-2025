* 審査機序図とは

審査機序図とは、建築確認申請における計画案件の法適合判定における情報処理のメカニズムを表現するための図式である。

* 審査機序図作成の目的
  + 建築確認申請における建築計画案の建築関連法令に対する法適合判定のメカニズムと申請時に提出される設計情報の要件を明らかにする。
  + 建築確認申請時の審査ソフトウェアに必要な機能要件を明らかにする。
* 審査機序図の活用
  + 審査機序図作成を作成する活動自体が、上記2項目の定義のための合意形成に寄与する。
  + ソフトウェアの機能要件の定義やその実装に役立てる。
  + 法改正時に従前の審査機序図の一部を修正することにより、設計情報要件と審査手続きの改正内容を明らかにする。
* 建築関連法令と審査機序図の関係
  + 審査機序図は、建築関連法令のうち、個別の建築計画の設計内容を規制する法文に対応して作成される。建築確認に対する行政の対応や手続き、審査者や建築士の資格等に関する法文には対象とならない。
  + 法文のうち何らかの規制内容を直接言明する条文をこの文書内で「規制文」と称する。
  + 条文の中から「～ならない。」で終わる文を検索することで規制文を拾い出すことができることが経験的にわかっている。審査機序図が対象とする規制文はその中で建築計画の立地条件や設計内容に関わるものである。
  + 建築関連法令の中心となるものは、建築基準法および建築基準法施行令であり。当面はこの二つの法令内の規制文への適否判定を目標とする。
  + 審査機序図の対象となる一つの規制文に対して少なくともその規制文への適否を最終の取得情報として表す審査機序図が一枚必要である。これを「適否判定審査機序図」という。
  + 上記のほか、規制文への適否に至るまでの途中段階の要素情報を導出するための一連の手続きを表現する審査機序図を作成することがある。このような審査機序図を「部分審査機序図」という。
* 審査機序図の図式仕様
  + 構成要素:
    一枚の審査機序図の構成要素は以下の5種類の図形である。

|  |  |
| --- | --- |
| ページタイトル | １枚の審査機序図が表す情報処理の概要を表現する。 |
| 情報を意味する図形: | 長方形で表現する。 その意味を示す簡潔な文字列を図形上に表示する。  (以後 特に審査機序図上の図形として表現される情報を意味する場合には [情報] と記述する。) |
| 処理を意味する図形 | 角を丸めた長方形で表現する。  その意味を示す簡潔な文字列を図形上に表示する。  (以後 特に審査機序図上の図形として表現される処理を意味する場合には。[処理]と記述する。) |
| [情報]から[処理]への矢印 | [情報]が[処理]に必要なインプットであることを示す。 |
| [処理]から[情報]への矢印 | [情報]が[処理]によって導きされたアウトプットであることを示す。 |

* + [情報]の種類

[情報]は、主体が持つ性質の種類によって以下のように分類される。

|  |  |  |
| --- | --- | --- |
| 命題真偽 | | 真か偽かが判定可能な事実  Ex.「○○の技術基準を満たすか否か」 |
| 区分情報 | | 既定のいくつかの区分の中のいずれかを示すもの  Ex.用途地域、遮音等級、建物用途コード |
| 数値 | | 大小関係に位置づけられる整数値または実数値  Ex.延べ面積 |
| 幾何学 概念 | 点 | Ex.天空率の計測点, 開口部の中心点 |
| 方向 | Ex.真北 |
| 線形状 | Ex.避難動線　建物と地盤面の接する線 |
| 面形状 | Ex.隣地境界 |
| ソリッド | Ex.柱/壁の形状, 延焼のおそれのある部分 |
| 集合定義 | | 主体と特定の関係にある他の主体がどれであるかを示す情報。明確に列挙可能な有限個を特定する情報。  Ex. 階(=主体) に属する外壁の集合  室(=主体) が属する階 |
| 視認情報 | | 上記の諸情報を得るために人による総合的な認識や判断を必要とする状況や状態  機械可読性のない情報全般  Ex.告示仕様に適合するか否かを判断するための断面詳細図 |

* [情報]として扱うべき情報

[情報]として表現される情報は、審査の対象となる個別の建築計画についての情報に限定される。法適合判定のためには、法令そのものの知識や都市計画情報のような公共データも必要であり、これらも情報であるといえるが、これらは個別の申請案件を処理するメカニズムの側に含まれるべきものであり、[情報]としては扱わない。これらはむしろ[処理]の中に含まれていると考えるべきである。

* [情報]の単位と捉え方

[情報]として図上で表現されるものは、主体とその性質とで構成される。
「主体」は、建築計画における物理的な物(壁、柱、など)や幾何学的な形状概念(床面積算定領域、延焼のおそれのある部分、敷地境界線,平均地盤面、など)具体的な計画案において数えることができるものである。
「性質」は、上記で定めた主体がもつ属性、他の主体との関係性、主体と特定の関係にある別の主体の特定情報である。

* [処理]に繋ぐべき[情報]

ひとつの[処理]に繋ぐべき[情報]は[処理] からのアウトプットとして導出する[情報]を得るために必要な情報である。その情報には、処理の対象としてどの主体に着眼するかを特定する情報と、各主体のどのような性質について知りたいのかを表す[情報]に大別され、両者を区別して別の[情報]の図形として表現する。
[処理]に繋げる[情報]は、コンピュータプログラム言語における関数とそのインプット引数の関係に似ているが、上述の点では異なっている。すなわち、プログラム言語の関数では、いくつかのオブジェクトのインスタンスを引数として渡し、受け取った関数の中でそれらのオブジェクトが保持している属性等を取得できるが、審査機序図におけるインプット[情報]は、処理対象が何であるかという[情報]とは別に処理対象のオブジェクトのどういう性質を処理に用いるかを [情報]として明示する。
